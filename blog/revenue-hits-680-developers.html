89| 89| 90| 90|

Today we're rolling out revenueholdings-license — a shared license key validation and rate limiting system — across the entire Revenue Holdings CLI tool ecosystem. This marks our transition from a pure open-source project to a sustainable, tiered product.

   115|   115|$ rh-schemaforge convert input.prisma --to drizzle

   117|   117|
   118|   118|# Free tier limit hit — upgrade prompt

Technical Design

140| 140| 141| 141|

We designed the license system with developer experience in mind:

142| 142| 143| 143|
    144| 144|
  • Offline-capable — JWT-based validation works without internet. License keys use RS256 signatures verified locally.
    • 145| 145|
  • 24-hour grace period — If your license server can't be reached, the CLI keeps working for 24 hours.
    • 146| 146|
  • SQLite rate limiting — Per-tool, per-day counters stored in ~/.revenueholdings/. No telemetry, no phoning home for usage tracking.
    • 147| 147|
  • Environment variable support — Set REVENUEHOLDINGS_LICENSE_KEY for headless CI environments.
    • 148| 148|
149| 149| 150| 150| 
# In CI or Docker
   151|   151|$ export REVENUEHOLDINGS_LICENSE_KEY=RH-PRO-XXXX-XXXX
   152|   152|$ deploydiff plan main..feature
   153|   153|✓ Infrastructure diff generated
154| 154| 155| 155|

Also Shipping Today: rh-envault Encrypt/Decrypt

168| 168| 169| 169|

Alongside the license rollout, Envault (envault 0.5.0+) now includes encrypt and decrypt commands — letting you encrypt individual environment values with AES-256 before committing them to version control:

170| 170| 171| 171| 
# Encrypt a sensitive value
   172|   172|$ rh-envault encrypt DB_PASSWORD

This pairs naturally with the license system — CI pipelines can use Pro-tier license keys to automatically decrypt environment variables during deployment.
